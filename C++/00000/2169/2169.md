# 2169. 로봇 조종하기
## 문제
숫자가 있는 2차원 배열이 주어지고, $(1, 1)$ 에서 시작해서 $(N, M)$으로 도착해야한다.</br> 왼쪽, 오른쪽, 아래쪽만 이동하고, 갔던 길을 가지 않으면서, 경로상에 있는 숫자 값 합의 최댓값을 구하는 문제

## 풀이
- 오른쪽과 아래쪽으로만 가는 문제의 변형.
- $N$번째 줄과 $1$번째 줄은 왼쪽을 갈 수 없다는 것이 포인트
	- 첫번째 줄은 $(1, 1)$에서 시작하기 때문에 왼쪽으로 갈 수 없다.
	- 마지막 줄은 왼쪽으로 가게되면, $(N, M)$으로 갈 방법이 없어진다.

1. 마지막 줄만 먼저 $(N,M)$에서 $(N, 1)$까지의 최댓값을 구한다.
2. $N-1$줄 부터 $2$번째줄까지 오른쪽으로만 갔을 때의 경로상 최댓값과 왼쪽으로만 갔을 때의 경로상 최댓값을 각각 구하고, </br>
 이후 DP 배열에는 각각의 위치에서 최댓값만 뽑아내어 저장한다. (23 ~ 38 Line)
	- 현재에서 아래로 내려갔을 때와 오른쪽 or 왼쪽으로 갔을 때의 값 중 큰 값을 배열의 값과 더해주면 된다.
3. $1$번째 줄이 되면 오른쪽에 대해서만 경로상 최댓값을 뽑아낸다.
	- 이 풀이에 의하면, N = 1일 때 고장나서(왜냐하면 3번을 돌린다는 것은 ${N}\geq{2}$ 이라는 것이기 때문) </br> 돌리면 안된다는 것 정도인데, 코드 잘 수정하면 안해도 될지도?
4. 답은 DP 배열의 $(0, 0)$이 된다.
